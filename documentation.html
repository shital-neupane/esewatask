Functional Specifications

The eSewa Hybrid Application is built to meet high-performance and security standards for mobile financial services.

Native App Requirements

  • Native Platforms: Android (Kotlin), iOS (Swift)
  • Android SDK: CompileSDK & TargetSDK: 35
  • iOS: Latest Swift version supported
  • UI/UX: Dynamic theming (Light/Dark) support with minimal, premium styling

Flutter Module Requirements

  • Pre-load Logic: Cache UUID from native; Dynamic load based on injected config
  • App Configurations: Includes ThemeMode, Colors, Spacing, and Typography
  • Architecture: Riverpod state management; Dio network library; API call logging; UUID header injection

Navigation Flow

1

Launch App

The app checks for a cached UUID. If found, it bypasses the login screen and immediately initializes the Flutter module. Otherwise, the Native Login screen is displayed.

2

Login Flow

Users enter credentials on a native screen with validation. On success, a random UUID is generated and stored securely. This UUID, along with theme configurations, is passed to the Flutter module upon initialization.

3

Flutter Interaction

The Flutter module loads the Product List. Users can tap to view Details. Clicking "Pay" triggers a return to the Native app with the selected product's payload.

Android Implementation File Library

App.kt

Subclasses Application to pre-warm the Flutter Engine and manage global dependencies. Orchestrates the initial application configuration dispatch.

MainActivity.kt

The primary entry point that hosts the Flutter UI. Manages the bidirectional MethodChannel for theme updates and payment actions.

LoginActivity.kt

Native Jetpack Compose screen for secure user authentication. Handles the transition logic from native to the Flutter-based main view.

AuthManager.kt

Handles user session lifecycle, including login, registration, and UUID generation. Uses encrypted storage for high-security persistence.

SecurityProvider.kt

Utility for initializing EncryptedSharedPreferences using MasterKeys (AES-256-GCM), ensuring sensitive data is protected at rest.

build.gradle.kts

Configuration script defining SDK versions (targeting 35) and managing dependencies for Jetpack Compose and Flutter integration.

iOS Implementation File Library

subiosApp.swift

The SwiftUI App entry point. Initializes the shared FlutterManager into the environment for global access across the UI.

FlutterManager.swift

Singleton manager for the iOS FlutterEngine and MethodChannels. Handles the "getAppConfig" calls and pushes real-time theme updates.

FlutterView.swift

A UIViewControllerRepresentable wrapper that allows the Flutter UI to be embedded seamlessly within the SwiftUI view hierarchy.

LoginView.swift

SwiftUI-based implementation of the secure login screen. Checks for existing session data before presenting the UI.

AuthManager.swift

iOS-specific session coordinator. Integrates with KeychainHelper for secure storage of the authentication UUID.

KeychainHelper.swift

Low-level security utility wrapping the iOS Security framework to provide simple, encrypted storage of session tokens.

Flutter Module File Library

main.dart

Defines the 'mainWithConfig' entry point. Bootstraps the app by receiving initial theme and session data from the native host.

models/app_config.dart

Freezed-powered data models for strict type-safe management of Theme, Spacing, and Typography tokens.

providers/config_provider.dart

Riverpod state notifier that manages the application configuration. Features local caching and live updates from Native.

services/api_client.dart

Advanced Dio implementation. Features interceptors for automatic header injection and secure logging for production.

services/native_channel_service.dart

The communication hub. Handles all asynchronous MethodChannel calls to Android and iOS for platform-specific actions.

screens/product_list_screen.dart

Responsive catalog UI that renders products using the dynamic theme tokens received from the native host.

Configuration models ensure that Flutter remains visually synchronized with its native host by consuming a standardized JSON schema for style tokens.

Auth & Session Management

esewasub/app/src/main/java/com/example/esewasub/LoginActivity.kt Android 
if (AuthManager.getActiveUser(this) != null) {
    startActivity(MainActivity.createIntent(this))
    finish()
}

The application implements session-safe bypassing. If a valid UUID is found in encrypted storage, the user is automatically transitioned to the main activity, skipping the login flow.

Channel & Integration

_dio.interceptors.add(InterceptorsWrapper(
    onRequest: (options, handler) {

        return handler.next(options);
    },
